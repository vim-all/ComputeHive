Project Image Flow

Build locally, then request a queued run with a hashed tar.gz artifact.

Choose a project directory and let ComputeHive generate Docker setup when it is missing. You can build the Docker image locally, then request a run so the app compresses the image to a tar.gz artifact, hashes it, uploads it to object storage, and writes the queue records into Redis.

Project Directory

Choose the project folder you want the app to Dockerize

ComputeHive checks for Docker setup in the selected project, creates Docker files when they are missing, builds the Docker image locally, and saves the image as a tar archive inside that same project root. Use the second action when you want the app to compress that image to a tar.gz artifact, hash it, upload it, and enqueue the Redis run request.

  • No container is created by this action.

Contributor Dashboard

Incoming run requests queued for local contributor execution

This surface is for contributors who offer their compute resources. Incoming run requests are loaded from Redis here, including the queued image tag, compressed artifact, and matching SHA-256 hash. The future Docker execution lifecycle stays grouped under Contributor Actions.
